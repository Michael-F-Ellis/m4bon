# Lessons Learned — Session 2026-06-29

## 1. Examples directory with full-length DSL transcriptions

Created `examples/` with 7 complete song transcriptions and a README. Key
takeaways for writing correct DSL by hand:

- **Measure boundaries first.** In 4/4, 4 beats per line. Break a melody into
  measures before writing groups — it's easy to forget and get "expected 1920
  ticks, got 3840" errors.
- **Dotted rhythm via cross-beat tie.** A dotted quarter + eighth is `e -d`
  (quarter E, then a beat with 2 eighths where `-` sustains E through the first
  slot). This is the only way to get durations that don't fit cleanly within
  one beat's subdivision slots.
- **6/8 compound meter:** `<numBeats>` is `timeNum/3` (2 dotted quarters per
  6/8 measure). Each beat group is 3 eighths — a single character `c` fills
  the whole dotted quarter.
- **Octave tracking with accidentals:** The DSL uses relative octave
  (closest-interval rule). Accidentals don't affect the octave calculation,
  so `#f` and `f` occupy the same octave window from the previous pitch.
- **Octave forcing:** Use `^`/`/` when a leap crosses the relative-octave
  threshold. Without them, `g /g` jumps down to G3 not G2 because the
  closest `g` from the previous `g` is the same octave.

## 3. Metronome inaudibility — conditional drum sample loading

The playback metronome uses wood block drum samples (MIDI notes 76/77 on
channel 10) via `wafPlayer.queueWaveTable`. The count-in uses oscillator
synthesis (`_scheduleClick`). Two separate paths — the count-in always worked,
but the playback clicks were silently dropped.

**Root cause:** `initSoftSynth` only loaded drum samples when
`this.metronomeOn` was true at init time. If the user's saved state had
metronome off, or if they toggled metronome after first play (when
`synthReady` prevents re-init), the drum presets were never loaded.
`getDrumPreset` returned `null`, and `scheduleNote` returned silently.

**Fix:** Load drum samples unconditionally in `initSoftSynth`. The samples
are tiny (< 10 KB each) — no reason to gate them behind a toggle. Also
removed the stale `getDrumPreset` call that was never reached because
`wafPlayer` check was bundled with `audioCtx`.

**Lesson:** When audio has two separate rendering paths (oscillator vs.
WebAudioFont), verify both independently. Don't assume "count-in works"
means the whole metronome works.

## 4. Comment lines in CSS table layout create anonymous table cells

Putting `<div class="m4bon-comment-line">` directly inside a `display: table`
container causes the CSS anonymous table-object model to wrap them in
anonymous `table-cell` boxes. These anonymous cells contribute to column
width calculations, meaning comment text length influenced horizontal spacing
between chord, note, and lyric columns.

**Fix:** Replaced CSS table layout with `display: flex; flex-direction: column`
on the measures container and `display: flex; flex-direction: row; gap: 4ch`
on each measure. Column spans use explicit `min-width: Nch` inline styles.
Comment divs are now block-level siblings of flex rows, not children of a
table element.

## 5. Right-justified measure numbers with fixed field width

The `buildMeasureCells` prefix was `fmt.Sprintf("%d:  ", measureNum)` which
produces variable-width output ("1:  " vs "10:  "), shifting note content
right when going from 1-digit to 2-digit measure numbers.

**Fix:** Added a `measureNumWidth` parameter computed from `numWidth(totalMeasures)`
and used `fmt.Sprintf("%*d:  ", measureNumWidth, measureNum)` for a right-justified,
fixed-width prefix. `numWidth` returns the decimal digit count of the last
measure number.

## 6. Stale `display: table-cell` CSS on flex children

After switching from CSS table layout to flexbox, the `.m4bon-chord-col`,
`.m4bon-note-col`, and `.m4bon-lyric-col` CSS classes still had
`display: table-cell` and `padding-right: 1.5em`. While `display: table-cell`
is ignored by flexbox, `padding-right` still applied, creating inconsistent
spacing. Even with correct fixed-width measure numbers, browser layout could
produce a visible shift.

**Fix:** Removed `display: table-cell` and `padding-right` from column classes.
Switched `white-space` from `nowrap` to `pre` to preserve explicit space cells.
Column spacing is now handled entirely by flex `gap: 4ch`.
