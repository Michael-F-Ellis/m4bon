## Key Topics

- Transcribing songs by ear vs. from reference
- The **kern** (Humdrum) format and kern.humdrum.org API
- kern → m4bon conversion: LUT-based octave resolution, beat packing, pickup handling
- Three modes of operation: local file, search+convert, list+pick

## DSL Lessons

1. **Inline comments don't exist** — `#` is the sharp accidental. Only `!` at line-start is a comment. Use `!` for full-line comments only.

2. **Beat subdivisions** create rhythm. `f--f` within one beat = 4 sixteenths (F, sustain, sustain, F). The sustain (`-`) between two of the same pitch creates a dotted effect within a beat — `g-g` = G for 2/3 of the beat + G for 1/3 (quarter + eighth in a dotted-quarter beat).

3. **The closest-interval rule** uses diatonic step distance, not semitones. Resolution is `o*7 + letterIndex`, scanning offsets -2..+2. From C, closest G is one octave DOWN (dist=3) not same octave (dist=4). This means octave modifiers are needed whenever a melody leaps wider than a 4th.

4. **Lyric tokens** map one-to-one with position slots in a measure. `_` glues multi-syllable words within one beat. `*` marks melisma (note continues the previous syllable). Rests and sustains need explicit lyric tokens too (`-` dash or `;` semicolon).

5. **The K/M header line** can't stand alone — it creates an empty measure 0. Always put K and M on the same line as the first notated measure.

## Kern Lessons

1. **kern.humdrum.org API** — the search form (`kssearch?s=t&keyword=X`) returns parseable HTML. Result rows have `ksdata?location=<path>&file=<file>&format=kern` links. Titles appear in `format=info` links ~2000 chars later. No JSON API but the CGI endpoints are consistent enough to automate.

2. **Search results** may include multiple variants of the same tune, and some hits are different songs that merely contain the search keyword. Use `--list` to disambiguate.

3. **Pickup measures** — notes before the first `=1` are anacrusis. The last measure should be shortened by the pickup's duration.

4. **Especially for Böhme/Erk collections** — titles are in ALL CAPS German and may differ from the common English title.

## Building an Automation Script

The kern→m4bon converter (`converters/kern2m4bon.py`) demonstrates:

- **Search** via `kssearch` HTML parsing (regex on format=kern links)
- **Fetch** via `ksdata` with location + file params
- **Convert** via the LUT-based octave resolution (49-entry table, no semitones)
- **Three CLI modes**: local file, search+convert, list+pick
- **Pickup handling**: detects pre-barline notes, shortens last measure
