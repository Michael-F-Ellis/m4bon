## Key Topics

## DSL Lessons

1. **Natural chord symbols are colored by default unless styled**:
   - The TUI and color-renderer styling architecture utilizes `StyleClass` identifiers to colorize chord symbols (`StyleSharp`, `StyleFlat`, etc.).
   - Natural chord symbols (e.g., `F⁶`, `Am7`) map to `StyleDefault` (no accidental) and should inherit the standard white/near-white text color rather than a muted grey to improve contrast and readability against dark backgrounds.

2. **Slash Chords styling**:
   - Slash chords style the root and the bass note independently (e.g. `B&/D` styles `B&` as a flat, `/` as default, and `D` as default).

## Deployment & Cache-Busting Lessons

1. **Legacy GitHub Pages Builder is highly unstable**:
   - The branch-based Jekyll builder on GitHub Pages can silently fail with generic errors ("Page build failed") for binary assets or compiled files (like updated `.wasm` packages), preventing any code changes from reaching the live site.
   - **Modern alternative**: Switch the repository to use official GitHub Actions (`actions/configure-pages`, `actions/upload-pages-artifact`, and `actions/deploy-pages`). This compiles the WASM binary and deploys the build directory directly, bypassing Git branch commits and Jekyll processing entirely.

2. **WebAssembly Client-side Cache Busting**:
   - Simply updating HTML or CSS query parameters (`app.css?v=X`) does not prevent browsers from permanently caching older `m4bon.wasm` files requested via a standard, query-less `fetch('m4bon.wasm')`.
   - **Robust Solution**: Programmatically extract the version query parameter from the main `<script>` tag inside JS and append it to the WASM fetch request dynamically:
     ```javascript
     const scriptEl = document.querySelector('script[src*="app.js"]');
     const version = scriptEl ? new URL(scriptEl.src, window.location.href).searchParams.get('v') : '0.23.1';
     const resp = await fetch(`m4bon.wasm?v=${version || '0.23.1'}`);
     ```
   - This guarantees that updating the version parameter in `index.html` cache-busts both the Javascript and the underlying WebAssembly binary synchronously.

3. **Actions Workflows & Sync Latency**:
   - When deploying via GitHub Actions, the runner remains active to monitor the final success status of the Pages deployment backend. This makes runs appear to take "several minutes", whereas branch deployments appeared instant because the terminal process detached immediately after the git push (with queuing happening silently on GitHub's side).

## Building & Verification

1. **Makefile comment syntax**:
   - Avoid nesting escaped backticks and string concatenations inside shell comments (e.g., `# Automatically run as part of ` + "`" + `make wasm` + "`" + `...`), as some models may interpret them as literal formatting commands instead of plain-text notes. Use simple quotes instead.
