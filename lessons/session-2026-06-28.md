# Lessons Learned — Session 2026-06-28

## 1. Look-ahead scheduler — two-path outcome

Implemented the look-ahead scheduler (plan `look-ahead-web-audio-scheduler.md`)
as a two-path approach:

**Playback-only:** 50ms scheduler with 200ms look-ahead. Keep-alive oscillator
(gain=0) prevents Safari auto-suspension.

**Recording:** Falls back to original "schedule all at once" — `MediaRecorder`
keeps the context alive, and front-loading node creation avoids incremental
`queueWaveTable` calls competing with capture.

Safari recording timing remains unreliable after multiple attempted fixes.
Chrome is now the recommended browser for the web TUI (README, AGENTS.md).

## 2. Dual chord root generators in `midi/generate.go`

`GenerateSMFWithOptions` (TUI) and `GenerateEventList` (web/WASM) both generate
chord root MIDI events. Both had the same one-beat duration bug. Fix applied to
both — always check both code paths when fixing MIDI generation.

## 3. Cursor highlight conflict during playback

`highlightCursorMeasure` (static blue highlight at editor cursor position)
conflicts with the moving `m4bon-playing` highlight during playback/recording.
**Fix:** guard `highlightCursorMeasure` to suppress the cursor class while
`isPlaying || isRecording`. Clear cursor on `play()` start, restore on
`stop()`/`onPlaybackEnd()`.

## 4. Leap note styling in web app

The `m4bon-leap-up`/`m4bon-leap-down` CSS classes were only emitted when
`asciiLeaps` was `true`, but the web app passes `asciiLeaps: false` (using
Unicode combining diacritics instead). **Fix:** added a shared `m4bon-leap`
class (bold + italic) that is always emitted alongside the directional class.
Directional classes keep only their text-decoration for ASCII mode; Unicode
diacritics handle the visual indicator in the default mode.

## 5. Chord root sustain via `:H` directive

Chord root MIDI notes always had 1-beat duration regardless of `-` sustain
tokens. Fixed by scanning ahead for consecutive `-` tokens and extending the
NoteOff accordingly. A chord like `C - G7 -` now sustains C for 2 beats and G7
for 2 beats.

## 6. Makefile conventions

- `make all` = build native binary + build WASM + run all tests
- `make serve` = serve `web/` on port 8087 (kills existing listener first)
- `make clean` = remove binaries, WASM, Go objects
