## 1. Newline as measure separator (DSL grammar change)

Eliminated `|` barlines from the DSL grammar. Measures are now separated by
newlines (one measure per line). End of file terminates the final measure.

**Key changes:**
- `SanitizeDSL` returns `[]string` (one element per non-empty, non-comment line)
- `ParseDSL(lines []string)` â€” iterates lines directly, each line tokenized and
  parsed as a measure
- `stripDirectives` removed â€” first line's directives handled by per-measure
  `scanMeasureDirectives`
- `splitMeasures` removed â€” lines already separated by newline
- `hasBarline` â†’ `len(lines) > 1`

**Backward compatibility note:** Test DSL strings that used `|` needed to be
rewritten with newlines. Multi-line `.dsl` test files that had `M4/4` on a
separate line from notes needed to be merged (directives now must be on the same
line as notation, since each line IS a measure).

**Test impact:** 17 `.dsl` files updated (`|` â†’ newlines). 8 files with
directives on separate lines merged. Several test DSL strings had `M4/4`
removed from partial-measure tests (previously `stripDirectives` extracted and
removed it before tokenization; now `scanMeasureDirectives` finds it and
triggers validation).

**Callers updated:** `m4bon.go` (Compile, RenderOptions), `cmd/m4bon/main.go`,
TUI code, all test helpers, WASM wrappers. Test helpers added `sanitizeLines()`
shim that splits on `|` for backward compatibility with old test strings.

## 2. Web app â€” barline removal and reformat regression

The web app editor was still inserting `|` barlines after the parser change.
Three locations updated in `web/app.js`:
- `highlightCursorMeasure`: count `\n` instead of `|` for measure index
- `reformatColumns`: restructured to split DSL by newlines directly instead of
  collapsing-then-splitting
- Removed `line += ' |'` from reformat output (measures joined by newlines only)

The auto-reformat was also too aggressive â€” firing on every keystroke with a
150ms debounce, it rewrote the textarea during typing. **Fix:** reformat only
fires on explicit actions: blur (leaving the textarea) or Ctrl+S / Cmd+S. Live
parsing and rendering still run on every keystroke.

## 3. WASM binary was stale

`make all` previously only did `build + test`, not `make wasm`. Added `wasm` to
`make all` dependencies so `go test ./...` and `make all` both rebuild the
world. Added `web/m4bon.wasm` to `make clean`. Also added `make serve` target
(port 8087, kills existing listener first).

## 4. Safari recording timing â€” pre-existing, not related

Safari's Web Audio timing becomes unreliable during recording (mic capture via
`MediaRecorder` + soft synth via `queueWaveTable`). This was masked before 0.17.1
because recording was entirely broken (suspended AudioContext, wrong MIME type).

**Root cause:** All notes are scheduled at once with absolute `startTime` values
relative to `audioCtx.currentTime`. Safari's audio clock can drift during
combined recording+playback, causing notes scheduled minutes ahead to land at
wrong times.

**Planned fix:** Look-ahead scheduler (see `plans/look-ahead-web-audio-scheduler.md`).
Schedule notes in 50ms chunks with a 200ms look-ahead window, re-polling
`audioCtx.currentTime` each cycle. Single code path for all browsers.

## 5. Makefile conventions

- `make all` = build native binary + build WASM + run all tests
- `make clean` = remove binaries, WASM, Go objects (separate, not in all)
- `make serve` = serve `web/` on port 8087 (kills existing listener first)
