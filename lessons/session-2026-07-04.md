## Key Topics

- Virtual Keypad Redesign: Replacing the complex multi-row grid virtual keyboard with a highly compact, horizontal, scrollable single-row toolbar.
- Standardized character ordering based on DSL notation sequence.
- Web UX improvements for touch-based devices.

## Virtual Keypad Toolbar Lessons

1. **Horizontal Scrollable Toolbar beats Grid Keyboards**:
   - Multi-row grids take up excessive vertical real estate on mobile/tablet devices, leaving very little space for viewing the rendered output columns (`CHORDS : NOTES : LYRICS`).
   - A single-row horizontal scrollable container (`overflow-x: auto`) with a hidden scrollbar (`scrollbar-width: none`) maintains full usability while preserving maximum vertical screen space.

2. **Sequential Ordering Mimics Syntax Entry**:
   - Arranging keys left-to-right to match the chronological order of character entry in the DSL sequence (`<chord open> <octaves> <accidentals> <pitch-letters> <chord close> <hold> <rest> <directive> <editing> <whitespace> <submit>`) minimizes thumb movement and increases cognitive layout predictability.
   - Exact sequence: `(`, `^`, `/`, `#`, `&`, `%`, `c`, `d`, `e`, `f`, `g`, `a`, `b`, `)`, `-`, `;`, `:`, `⌫`, `Space`, `⏎`.

3. **Styling and Color Association**:
   - Matching keypad button text colors with the actual rendered notation colors (e.g., pitches as accent, sharps as red, flats as blue, rests/sustains/chords as grey, directives as orange) creates a strong spatial-visual link for users learning the DSL.
   - Buttons should use `flex: 0 0 auto` with small, explicit padding and min-width to ensure they do not shrink when rendered inside a flexbox toolbar.

4. **LocalStorage Name Consistency**:
   - When loading and saving persistent application states, ensure key names match precisely (e.g., using hyphens consistently like `m4bon-keypad-active` rather than mixing with underscores like `m4bon_keypad_active`), to avoid state discrepancies on page reload.
